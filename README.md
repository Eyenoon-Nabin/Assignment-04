1. getElementById, getElementsByClassName, এবং querySelector/querySelectorAll এর মধ্যে পার্থক্য কী?

*getElementById: এটা দিয়ে শুধু একটা নির্দিষ্ট এলিমেন্টকে তার ইউনিক ID দিয়ে ধরা হয়।

*getElementsByClassName: এটা দিয়ে একই Class যুক্ত অনেকগুলো এলিমেন্টকে একসাথে ধরা হয়।

*querySelector: এটা CSS সিলেক্টরের মতো কাজ করে, কিন্তু শুধু প্রথম যে এলিমেন্টটাকে পাবে, তাকেই ধরবে।

q*uerySelectorAll: এটাও CSS সিলেক্টরের মতো কাজ করে, কিন্তু ম্যাচ হওয়া সবগুলোকে একসাথে ধরে একটা লিস্ট (NodeList) হিসেবে দেয়।

2. DOM-এ নতুন এলিমেন্ট কীভাবে তৈরি এবং ইনসার্ট করতে হয়?

প্রথমে document.createElement('ট্যাগের নাম') দিয়ে একটা নতুন ট্যাগ বানাতে হয়।

ইনসার্ট করা: এরপর সেই তৈরি করা এলিমেন্টকে appendChild() বা append() ব্যবহার করে কোনো একটা Parent (বড়) এলিমেন্টের ভেতরে ঢুকিয়ে দিতে হয়।

3. Event Bubbling কী এবং এটা কীভাবে কাজ করে?

Event Bubbling হলো পানির তরঙ্গের মতো। তুমি যদি কোনো Child এলিমেন্টে (যেমন বাটনে) ক্লিক করি, সেই ক্লিক ইভেন্টটা শুধু তার মধ্যেই সীমাবদ্ধ থাকে না। সেটা বুদবুদের মতো উপরের দিকে উঠতে থাকে এবং তার Parent, Grandparent হয়ে একদম body পর্যন্ত চলে যায়।

অর্থাৎ, ভেতরের কোনো কিছুতে ক্লিক করলে, বাইরের এলিমেন্টগুলোও মনে করে যে তাদের ওপরেও ক্লিক পড়েছে।

4. Event Delegation কী এবং এটা কেন উপকারী?

 আলাদা বাটনে আলাদা ইভেন্ট লিসেনার না বসিয়ে, মাত্র একটা ইভেন্ট লিসেনার বসানোকেই Event Delegation বলে।
এটা ওয়েবসাইটের মেমোরি বাঁচায়। আর ভবিষ্যতে যদি নতুন কোনো বাটন যোগ হয়, তাহলেও আলাদা করে ইভেন্ট লিসেনার বসাতে হয় না, Parent একাই সব সামলে নেয়।

5. preventDefault() এবং stopPropagation() এর মধ্যে পার্থক্য কী?

preventDefault(): এটা ব্রাউজারের জন্মগত স্বভাবকে থামিয়ে দেয়। যেমন: ফর্মে সাবমিট বাটনে ক্লিক করলে পেজ রিলোড হয়ে যায়, preventDefault() দিলে সেই রিলোড হওয়া বন্ধ হয়।

stopPropagation():"Event Bubbling"—কে থামায়। অর্থাৎ, ইভেন্টকে Parent-এর দিকে যেতে দেয় না, যেখানে ক্লিক পড়েছে শুধু সেখানেই আটকে রাখে।
